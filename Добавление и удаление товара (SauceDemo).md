# 1. Добавление товара (кнопка «Add to cart»)
## Позитивные
- [x] Клик мышью по кнопке -> Кнопка заменяется на «Remove».
- [x] Клик мышью по кнопке -> На иконке корзины появляется красный бейдж со счетчиком (или цифра увеличивается на 1).
## Негативные:
- [x] Быстрое многократное нажатие на кнопку (даблклик) -> Добавляется только 1 экземпляр товара.
## Cистемные
- [x] Обновление страницы (F5) после добавления -> Кнопка остается в состоянии «Remove», счетчик корзины не сбрасывается.
# 2. Удаление товара (кнопка «Remove»)
## Позитивные
- [x] Клик мышью по кнопке -> Кнопка заменяется на «Add to cart».
- [x] Клик мышью по кнопке (когда в корзине >1 товара) -> Счетчик на корзине уменьшается на 1.
- [x] Удаление последнего/единственного товара -> Красный бейдж со счетчиком полностью исчезает (граничные значения).
# 3. Сортировка товаров
## Позитивные
- [x] Открытие страницы каталога (состояние по умолчанию) -> В дропдауне выбрано Name (A to Z). Карточки отсортированы по алфавиту от А до Я.
- [x] Выбор Name (Z to A) -> Карточки перестраиваются по алфавиту в обратном порядке.
- [x] Выбор Price (low to high) -> Цена каждой следующей карточки больше или равна предыдущей
- [x] Выбор Price (high to low) -> Цена каждой следующей карточки меньше или равна предыдущей
## Граничные
- [x] Сортировка по цене при наличии товаров с одинаковой стоимостью -> Товары с идентичной ценой сортируются по алфавиту.
## Системные
- [x] Обновление страницы после применения любой недефолтной сортировки -> Выбранный тип сортировки сохраняется, карточки остаются в заданном порядке
# 4. Проверки UI
- [x] Изменение окна браузера до размеров мобильного телефона -> Кнопки добавления/удаления не выходят за пределы экрана и не перекрывают цену. Дропдаун сортировки остается кликабельным, текст внутри не обрезается и не наезжает на другие элементы.
